# 開發日誌（2026-04-02）

## 本次目標
- 把「選項機率權重」改成「純故事 + 附近要素」驅動。
- 修正換區/傳送門引導，避免突兀與連續重複塞選項。
- 補強多人討論串按鈕安全，避免別人按壞他人流程。

## 主要改動

### 1) 選項引擎：移除機率權重，改成連貫分數排序
檔案：`bot.js`

- 移除 `RISK_CATEGORY_WEIGHTS` 權重抽選流程。
- `normalizeEventChoices(...)` 改為：
  - 先組合上下文訊號（目前故事尾段、上一行動、地區篇章進度、附近要素）。
  - 對 7 個候選計算 `computeChoiceContinuityScore(...)`。
  - 依分數排序選 5 個，並保留基本多樣性（同類別不過度集中）。
- 仍保留關鍵系統項目的保底（如傳送門/許願池/市場），但不再靠機率抽到。

### 2) 系統選項注入：從「機率觸發」改為「條件觸發」
檔案：`storyteller.js`

- 移除 `SYSTEM_OPTION_CHANCE` 常數與對應 random gate。
- `injectPortalChoice` / `injectWishPoolChoice` / `injectMarketChoices` / `injectScratchLotteryChoice` / `injectMentorSparChoice` 全部改為：
  - 只看「故事訊號 + 附近可互動要素 + 必要強制條件」決定是否注入。
- 市場注入策略調整：
  - 非強制時，依地區難度與新手遮罩，傾向對應市場類型，不再隨機挑市場選項。

### 3) 傳送門引導：加入節奏冷卻與跨區訊號判斷
檔案：`bot.js`

- 新增 `PORTAL_RESHOW_COOLDOWN_TURNS`（預設 2）。
- 每個地區曝光狀態新增 `portalLastShownTurn`。
- `ensurePortalChoiceAvailability(...)` 改為：
  - 看故事是否出現「傳送門/跨區」線索。
  - 看是否主線明確要求跨區。
  - 看地區篇章是否接近完成。
  - 套用冷卻，避免每回合重複推同種傳送門引導。

### 4) 市場保底（bot 端）去隨機化
檔案：`bot.js`

- `createGuaranteedMarketChoice(...)`：
  - 由「新手期 + 地區難度」決定偏向 Renaiss 或 Digital，不再 random。
- `ensureMarketChoiceAvailability(...)`：
  - 移除 `Math.random() > 0.35` 的擋門。
  - 改為「附近是否有市場要素 + 劇情是否有市場語境 + 節奏冷卻」。

### 5) 討論串按鈕安全修正
檔案：`bot.js`

- `rejectIfNotLatestThread(...)`：
  - 只會清理「同擁有者」的舊按鈕，不會清掉其他玩家訊息按鈕。
- 追加 thread owner 檢查到幾個入口按鈕流程，避免跨人操作。

## 行為結果（預期）
- 選項不再是硬塞機率感，而是以當下故事和地區上下文為主。
- 同地區停留過久時，會更自然引導到跨區，不會突兀硬跳。
- 公開討論串可看，但非擁有者更難干擾他人流程。

## 相容性與設定
- 新增可調參數：
  - `PORTAL_RESHOW_COOLDOWN_TURNS`（預設 `2`）
- 其餘改動以既有資料結構相容為前提，不需手動 migration。
